4. How We Use Your Data

4.1 Core Features

  • OCR Scanning: Text extraction from receipts (processed locally, images NOT uploaded to our servers)
  • Multi-Currency OCR: Automatic detection of 14 currencies (EUR, USD, GBP, CHF, JPY, CAD, AUD, CNY, INR, BRL, MAD, KRW, RUB, MXN)
    • Detection via symbols/patterns on receipt (local OCR)
    • Device language fallback if no currency detected
  • Split Calculations: Equal or custom split algorithms
  • Synchronization: Cloud Firestore to access your data across multiple devices
  • Push Notifications: Firebase Cloud Messaging for budget alerts, new group expenses, reimbursements
  • Group Invitations: Generation of links/QR codes with temporary tokens (30-day expiration)

4.2 Service Improvement

  • Crashlytics: Bug detection and correction
  • Firebase Analytics: Anonymized usage statistics (user count, popular features)
  • A/B Testing: User interface optimization

6. Data Security

6.1 Technical Measures

  • Encryption in transit: TLS/SSL for all communications with Firebase
  • Encryption at rest: Data stored encrypted on Google Cloud servers
  • Secure authentication: OAuth 2.0 (Google Sign-In), bcrypt for passwords
  • Firestore Rules: Strict access restrictions:
    • Users: Read/write own data only (/users/{userId})
    • Groups: Public read for ALL authenticated users (required for QR code/link invitations), write only for members
    • Expenses: Access only for group members or solo group owner
    • Invite tokens: Public read for ALL users (anonymous access for invitation links), authenticated write/delete
    • Reports: Read only by reporter, create only
    • Settlements/Disputes: Members only

Security Trade-off: Group basic info (name, emoji, participants) is publicly readable by authenticated users to enable QR code/link invitations. This is a necessary trade-off for invitation functionality.

  • Invitation tokens: Automatic expiration after 30 days
  • Server-side validation: Cloud Functions for sensitive operations (disputes, settlements)

6.2 Organizational Measures

  • Data access limited to authorized developers only
  • Database access logs
  • Regular security audits
  • Frequent dependency updates (security patch fixes)

6.3 No Absolute Guarantee

Important: No system is 100% secure. We do our best to protect your data, but we cannot guarantee absolute security.

Security Tips:

  • Use a strong password (if email login)
  • Don't share your password
  • Sign out on shared devices

13. Specific Data Processing

13.1 Receipt Photos (OCR)

Important: Receipt photos are processed locally on your device via Google ML Kit.

  • No photos are uploaded to our servers
  • Only extracted data (text) is stored in Firestore
  • Photos are deleted immediately after processing

13.2 Group Data (Visibility)

Data visible to group members:

  • Your name
  • Your shared expenses in the group
  • Your balances (amounts owed/receivable)
  • Your status (admin or member)

Data NOT visible:

  • Your email (unless manually shared)
  • Your personal expenses (solo mode)
  • Your budgets
  • Your other groups

13.3 Disputes and Conflicts

When you dispute an expense:

Recorded data:

  • Your name (visible to all)
  • Dispute reason (free text, max 500 characters)
  • Dispute date
  • Status (open, resolved)
  • Resolved by (admin name + date if resolved)

Visibility:

  • All group members see the dispute
  • History kept in expense (even after resolution)

Cloud Functions:

  • disputeExpense: Records dispute + notifies members (NO moderation)
  • resolveDispute: Marks as resolved + notifies

Important: Dispute reasons are NOT moderated. Basic local validation only (forbidden words, length check).

Legal Basis: Consent (voluntary action) - Art. 6(1)(a) GDPR

13.4 Settlements/Reimbursements

When you mark a debt as paid:

Recorded data:

  • Reimbursed amount
  • Payment method (cash, revolut, lydia, bank transfer)
  • From who ‚Üí To who (IDs + names)
  • Reimbursement date
  • Related group

Visibility:

  • All group members see the settlement
  • Used to automatically recalculate balances

Cloud Functions:

  • notifySettlement: Notifies both parties + group

Warning:

  • No proof of payment stored (no screenshot, no bank reference)
  • Based on trust between members
  • Use traceable methods (Revolut, bank transfer) for security

Legal Basis: Consent (voluntary action) - Art. 6(1)(a) GDPR

13.5 QR/Link Invitations

Invitation tokens:

  • Randomly generated (timestamp)
  • Stored in Firestore (invite_tokens/{token})
  • Expiration after 30 days
  • One token = one group only
  • Automatic cleanup of expired tokens

Security: Only share links/QR with trusted people. Anyone with the link can join the group for 30 days.

13.6 Content Moderation

To protect all users, we use:

Local moderation only:

  • Forbidden words list (insults, spam)
  • Length verification (3-100 characters)

API moderation NOT used:

  • Google Perspective API is NOT used in production
  • Only local validation applied
  • No external API calls for moderation

Legal Basis: Legitimate interest (community safety) - Art. 6(1)(f) GDPR

13.7 Receipt Templates

To improve OCR scan accuracy, ShareSpend memorizes:

Data stored locally (SharedPreferences):

  • Store name (optional)
  • Average position of item names (X pixels)
  • Average position of prices (X pixels)
  • Average vertical spacing between lines
  • Number of scanned items
  • Confidence score (scan quality)

Usage:

  • Pre-fill OCR parameters for subsequent scans
  • Improve automatic detection
  • Maximum 20 templates stored (FIFO)

This data remains on your device, never sent to our servers.

Legal Basis: Legitimate interest (UX improvement) - Art. 6(1)(f) GDPR

13.9 Inappropriate Content Reporting

You can report:

Reportable content:

  • Group (offensive name, spam)
  • Expense (fraudulent amount, inappropriate description)
  • User (abusive behavior)

Recorded data:

  • Your ID (reportedBy)
  • Content type (contentType)
  • Content ID (contentId)
  • Reason (reason: spam, harassment, fraud, inappropriate)
  • Optional details (details: free text)
  • Status (pending, reviewed, resolved)
  • Report date

Processing:

  • Stored in Firestore reports/{reportId}
  • Manually reviewed by moderators
  • No automatic response (variable delay)

User blocking:

  • You can block a user (stored in users/{yourId}/blockedUsers)
  • You will no longer see their groups/expenses
  • They can still see yours (no automatic reciprocal blocking)

Legal Basis: Legitimate interest (community safety) - Art. 6(1)(f) GDPR

13.10 Product Database (OCR Improvement)

To improve automatic item recognition:

Data used:

  • 5000+ common products (name, average price, categories)
  • OCR patterns (name variants, common errors)

Storage:

  • Local assets (embedded in app, no server sync)
  • Manual update (app releases only)

Usage:

  • Progressive matching (3 passes: exact, OCR variants, fuzzy)
  • Automatic OCR error correction
  • Category suggestions based on product name

Your purchases are NOT collected to enrich the database

Legal Basis: Legitimate interest (UX improvement) - Art. 6(1)(f) GDPR
